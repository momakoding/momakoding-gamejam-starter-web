LOGO

周末 48 小时，一个人（加几个 AI）也能做出能玩、能扫码、能直接上线的网页游戏。Vue 管菜单 UI，Phaser 管玩法物理，AI 照登记表改数值——你只管想好不好玩。

WHO IT'S FOR / 它为谁而造
● 程序员： fork 之后跳过搭架子的半天，菜单、暂停、路由、构建链全配好，第一行代码就写在玩法里。
● 策划 / 产品 / 设计师： 不用学 Phaser，不用读源码。改占位文案、调数值、靠 AI 把「手感」翻译成代码，当天就能在浏览器里玩到自己的想法。
● 人 + AI 混编团队： spec-game.md 当设计契约，AGENTS.md 当工程契约。AI 改代码前先对账，人类负责定义体验，分工不打架。
WHY THIS STARTER / 为什么选它？
Q: UI、菜单、设置面板，每次都要在 Canvas 里手搓？
A: 48 小时的体力账本里，最贵的从来不是玩法，是 UI。Canvas 手搓按钮、菜单、设置面板要重复造轮子，没有热更新、没有焦点管理。把 DOM 的活交回 Vue 3 + Tailwind，Phaser 专心做渲染、物理、输入循环——分工清楚，迭代速度直接翻倍。
Q: 多人或 AI 协作时，代码总被改乱怎么办？
A: 三层物理隔离 + 分支约定。engine/ 是引擎层、contents/ 是游戏世界、runtime/ 是 Vue 侧粘合层，导入方向单向不可逆。具体玩法只在 games/ 或 demo/* 分支推进，main 始终保持可 fork 的纯净底座。换游戏不用重搭架子，协作者也不会互踩。
Q: 队里的策划或 AI 不会 Phaser，怎么参与？
A: AGENTS.md 把场景键、事件键、资源键、调参常量登记成一张活表。非程序员或 AI 只要改 contents/constants.ts 里的重力、初速度、关卡数值就能调手感；改 game-info/ 里的标题、说明、团队信息，主页和关于页同步刷新。完全绕开 Phaser 的类继承细节。
Q: 我完全不会写代码，也能用吗？
A: 能。配合 Cursor、Claude Code、Kilo Code 这类 AI 助手，你只描述「跳起来要更轻盈」「金币减到 5 个」，它来改代码。脚手架还会把你的玩法描述沉淀进 docs/vibe/spec-game.md，作为后续所有迭代的唯一设计源——AI 不会每次都从零理解你的脑洞。

WORKFLOW / 典型工作流

不会写代码也没关系。典型的迭代节奏是这样的：

① 描述想法 → ② AI 改代码 → ③ 浏览器试玩 → ④ 继续微调

跑起来后，优先改这四处占位内容：

src/contents/game-info/game-meta.ts — 游戏名称 / 副标题
src/contents/game-info/how-to-play.md — 玩法介绍正文
src/contents/game-info/team.ts — 团队名称 / 成员
src/style.css @theme { ... } — 整体配色
→ 上手指南 (CN) — 完整新手流程，含常见失误与 FAQ
FEATURES / 特性 ● 开箱即用： 标题菜单、占位关卡、暂停/恢复/退出、Hash 路由全部就绪。pnpm i && pnpm dev 即开始迭代玩法，省下半天搭架子的成本。
● 分层架构： engine / contents / runtime / pages 单向依赖，main 分支永远纯净。重构不会牵一发动全身。
● AI 协作友好： AGENTS.md 作为机制契约：登记表 + Read-before-write checklist + 决策日志，让 AI 在改代码前先对账，少幻觉、少返工。
● 占位优先： 无美术也能跑：纯色矩形即占位，game-info/ 集中管理标题、说明、团队信息。先把玩法跑通，美术后补，节奏不被资源卡死。
● 可路演 / 可上线： pnpm build 出的是真生产物，不是 demo 残留。Hash 路由适配 GitHub Pages、itch.io、Vercel 等静态托管，无需服务端配置。路演时扔一个 URL 就行——不用打包 exe，不用现场连本地，评委扫码即玩，作品上线即归档。
TECH STACK / 技术栈
UI : Vue 3 (SFC + Composition API)
Engine : Phaser 4 (Arcade Physics)
Styling : Tailwind CSS v4
Build : Vite 8 + TypeScript Strict Mode
ARCHITECTURE / 架构
src/runtime/ - UI 无关的 Phaser 包装器 (GameShell)
src/games/ - 具体的游戏内容与关卡
src/pages/ - Vue 构建的菜单与 UI 系统

快速开始：

$ pnpm install
$ pnpm dev # 启动 http://localhost:5173/
